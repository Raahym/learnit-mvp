# LearnIt UI/UX System

## Design Direction

LearnIt should feel like a serious academic command center, not a generic AI SaaS template.

## Rules

- Prefer truthful empty states over decorative placeholders.
- Show evidence labels for recommendations.
- Keep dashboard density high but readable.
- Do not show disabled dream navigation.
- Use restrained cards for repeated items and tool surfaces.
- Avoid fake metrics.
- Make missing setup obvious and actionable.

## Page Expectations

- Today: state, primary action, evidence, setup checklist, real activity.
- Subjects: subjects, topics, exam dates, targets.
- Plan: saved tasks and task creation.
- Quizzes: diagnostic and generated quiz attempts.
- Library: materials, upload state, processing state.
- Progress: readiness confidence and evidence.
- Mistakes: repeated weak patterns.
- Settings: account and preferences.
